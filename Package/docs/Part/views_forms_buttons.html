Form Fields and collections of fields (or Field-list called a Fist)

HTML5 no longer requires the <form> tag, and is not used in EpicMvc. A quick HTML tag to display a collection of Fields and thier attributes is the tag: <e-fist fist="FistName">. By default, EpicMvc will attempt to draw each field defined for the Fist using the part name 'fist_default' (a generic HTML part that is included with EpicMvc is in the Proto package: Proto/Part/fist_default.html.

These fields, when properly built as fist_default.html does, are 2-way bound to the Base Model called 'Fist' which implements all of the validation and filtering. When a clickable action occurs, the Controller can ask for this Fist to be validated, and pass those fields to a Model.action.

Assuming you have created a fist called 'Login' with any number of fields, and wish to create a button ("Login") the HTML might look like this:

Page/login.html
	<div class="container">
		<e-fist fist="Login"/>
	</div>
	<button e-click="login" value="Login" class="btn btn-primary" />
</fist>

To create a fist and its fields or edit attributes of those fields, see the topic: Creating Forms and Editing FISTs.

Custom HTML

Custom HTML for a Fist can be used by specifying the HTML View Part within the attribute: part="" on the <e-fist/> tag or the part: attribute on the FIST definition. When using a custom View Part to display a Fist, EpicMvc will use the "View" Model to populate the Field information. The "View" Model exposes 'Control' and 'Field' tables that can be looped on to display every Control or Field that is defined for that Form. The default is to expose 'Control' but can be changed with via="Field" or via:. Control will invoke your 'part' for each field in the order listed in the FIST definition, whereas Field will call the part once and each field is accessable by name to allow manual placement with any other HTML in any place desired.

Using a custom View Part to display the Fist, the above example could be written as follows:

Example

Page/login.html
	<e-fist fist="Login" part="fist_login" via="Field" /> <!-- uses Part/fist_login.html -->
	<e-form_action action="login" value="Login" class="btn btn-primary" />
Part/fist_login.html
	<div>
		<e-foreach table="Field/UserName">
			<input
				e-event="Fist:&UserName/fistNm;:&UserName/fieldNm;:keyup-focus-blur-change"
				type="&UserName/type;"
				name="&UserName/name;"
				value="&UserName/value;"
				?required="&UserName/req#?%;
				placeholder="&UserName/label;">
		</e-foreach>
	</div>
	<div>
		<e-foreach table="Field/Password">
			<input
				e-event="Fist:&UserName/fistNm;:&UserName/fieldNm;:keyup-focus-blur-change"
				type="&Password/type;"
				name="&Password/name;"
				value="&Password/value;"
				?required="&UserName/req#?%;
				placeholder="&Password/label;">
		</e-foreach>
	</div>
