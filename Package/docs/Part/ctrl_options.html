OPTIONS

OPTIONS is a top level key of the Controller structure and is used to define various settings and defaults for the app.

Example

window.EpicMvc.app$myPkg=
	OPTIONS:
		login: 		flow: 'anon'
		template:	default: 'logged_out', popup: 'modal_popup', pop:'pop'
		settings:	group: 'main_group', show_issues: 'inline'

login:

The login: flow: option is used to tell the Controller which flow to start the application under.

template:

The template: option is used to define the file name of the default HTML template, default:, along with any other Modal templates that will be used in the application. In the example above, popup is the name for the modal that will be referenced by the Controller within the Flow, and 'modal_popup' is the name of the file on disk modal_popup.tmpl.html that holds the HTML for that modal.

settings:

The settings: option is used to define which Fist Group, group:, this controller should be dealing with and weather or not to display form issues 'inline', show_issues: 'inline'. Inline issues are displayed next to the field that has an issue as opposed to a generic issue message at the top of the page.
