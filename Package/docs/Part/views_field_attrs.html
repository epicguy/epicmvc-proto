Field Attributes

Attribute Required Example Description
db_nm yes 'username' The database name of a field
type yes 'textarea' The type of the form field to display. See below for options.
label no 'Confirm Password' The label to place on a form field
req no true Field must not be blank if set to true
req_text no 'Password is required' The text to display if the field is required, but left empty
validate no 'regexp' The name of the validation filter to run. See below for options.
validate_expr no '[0-9]{3,4}' An expression passed to the validation filter
issue_text no 'Password must be at least 6 characters long' The text to display if there is an error validating the field
max_len no 8 Return an error if the field value is larger than the max_len
h2h no 'trimspaces:lower_case' The spec that is passed to the H2H Pre-filter and H2H Generic Filter. See below for options.
d2h no 'phone' The name of the D2H filter to run. See below for options.
d2h_expr no 'international' An expression that gets passed to the D2H filter
h2d no 'date' The name of the H2D filter to run. See below for options
h2d_expr no 'yyyy-mm-dd' An expression that gets passed to the H2D filter
cdata no [1,2,3,4,5,6,7] Data used to populate multiple choice field values. e.g. pulldown:array

Field Attribute Options

type text Single line text box
textarea Multi-line textbox
radio:array Radio buttons populated with the array from cdata attribute
radio:custom Radio buttons populated by the Model's fistGetFieldChoices() method
yesno Checkbox
pulldown:array Pulldown populated by an array of data from the cdata attribute
pulldown:custom Pulldown populated by the Model's fistGetFieldChoices() method
pulldown:wist Pulldown populated by the Wist Model; See cdata for details.
pulldown:json_like Pulldown populated with a JSON like key/value structure from the cdata atrribute.
cdata [val1, val2,...] Used by pulldown:array and radio:array. List of values that will be used as the db value and display in the HTML
[[val, opt],[val2,opt2]...] Used by pulldown:array and radio:array. List of lists. Each sub-list will have a val that is used for the db, and an opt to display in the HTML
'wist_grp:wist_nm:val_nm:opt_nm' Used with pulldown:wist type. Epic will call the 'Wist' Model with a table name of 'wist_grp:wist_nm' and use the 'val_nm' and 'opt_nm' of the Wist data to populate the options and values for a multiple choice input.
'yes_value' Used with yesno type. Epic will assign the 'yes_value' when the checkbox is checked. Defaults to 1
validate phone Validates the value has exactly 10 digits
zip Using: validate_expr: '5or9' validates the value is 5 or 9 digits
choice Validates the value is a valid choice. If validate_expr:true, then the value must not be the first choice.
email Validates a valid email format
regexp Validate value matches the regexp on the validate_expr attribute
confirm Validate value matches the field name defined in the validate_expr attribute
h2h trim_spaces Removes all whitespace
digits_only Removes non-digit characters
lower_case Converts to lower case
upper_case Converts to upper case
d2h phone Removes non-digits; Converts to format (xxx) xxx-xxxx
date Converts to format m/d/y
blank_is_zero Converts to 0 if value is empty
h2d phone Removes non-digits
zero_is_blank Converts to empty string if value is blank
