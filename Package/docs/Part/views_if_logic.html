Conditional Logic Using IF Statements

<epic:if> tags are used to surround a set of markup/content lines so that they may only be included on certain conditions. Frequently the conditions are buried in some complex logic, and this causes a few issues. One issue is that the various conditions being checked are spread throughout the HTML. EpicMvc allows you to collect all this logic at the top of a file, and supply names for them, to be used by name later in the file. This way you also only need to modify the condition in one place for it to be used elsewhere. Also, you can refer to the same name more than once, which cuts down on complexity, and lets the computer calculate the result only one time, for use multiple times in the same http request.

There are five tags dealing with different types of logical situations: <epic:if> , <epic:if_any> , <epic:if_all> , <epic:if_true> , <epic:if_false>

Typically, a comparison has two operands. If you want to know if a variable is equal to a certain value, then you have (a) a left side, (b) an operator, and (c) a right side: left='variable' op='eq' 'right='string value'.

In EpicMvc this is done as follows: 

<epic:if left="&Items/quantity;" op="eq" right="1">
	<div>This quantity is 1!</div> # This HTML would be output
</if>

Suppose you need to know if the quantity is both >1 and <5? You might be interested to know that the tag <epic:if> is really a synonym for <epic:if_all> . There is a corresponding <epic:if_any> .

Therefore, to code for >1 and <5,  you would do this: 

<epic:if_all left="&Items/quantity;" right="1" op="gt" right="5" op="lt">
	<div>This quantity is between 2 and 5!</div> # This HTML would be output
</epic:if_all>

Many times, one wishes to do one thing when true, and another when the 'if' is false (much like the if-then-else type logic.) The best way to effect this, is to create a label for the operation, and then use the label in an if_true and if_false fashion, like this: 

<epic:if_all name="one_to_five" left="&Items/quantity;" right="1" op="ge" right="5" op="le" />

Now, you can use the 'name' of 'one_to_five' in these two tags: 

<epic:if_true name="one_to_five">
	<p>There are one to five items here</p>
</epic:if_true>

<epic:if_false name="one_to_five">
	<p>There are not one-to-five items here.</p>
</epic:if_false>

The operators that you can use are as follows:

Operator
Meaning
eq equal to
ne not equal to
gt greater than
ge greater than or equal to
lt less than
le less than or equal to
set is this variable set with a non-empty value?
not_set is this variable empty?
table_is_empty does this table have no rows?
table_is_not_empty does this table have at least one row?
