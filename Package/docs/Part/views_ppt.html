Frames, Layouts, Pages, and Parts

'Frames', 'Layouts', 'Pages', and 'Parts' are HTML files located on disk, that contain valid HTML excluding the traditional <!DOCTYPE html>,<html>, <head>, <meta> and <body> tags. 'Frames' and 'Layouts' are saved as Layout/*.html, 'Pages' as Page/*.html and 'Parts' as Part/*.html.

Parts

EpicMvc 'Parts' are the most basic components of the 'View'. A 'Part' could contain as little as a single <br/> element, or include most of the HTML for the application. However, the idea is to design 'Parts' as meaningful and modularized pieces of the HTML that can be included or looped on elsewhere in the 'View'.

To include the HTML defined in the 'Part' simply use the element <e-part part="part_file_name_without_the_html_extension">. For example part="login" corresponds to the Part/login.html file on disk. EpicMvc will then replace that element with the actual HTML from the Part during render. The 'Part' element can be included in any 'Page', 'Element', 'Frame' or even another 'Part'.

Pages

An EpicMvc 'Page' is the main component of the 'View' and is specified using the <e-page /> element. Only one 'Page' is displayed at a time, (although, with a popup a previous page may appear in the backgound) and it is composed of HTML and any 'Parts' that are included.

Layouts

An EpicMvc 'Layout' is HTML that includes a single EpicMvc 'Page' using the HTML element <e-page /> along with any number 'Parts' and any amount of HTML. It will be rendered "around" the Page, based on where you plae the e-page element in this HTML file. Layouts are specified in the Controller file using the hash layout: 'layout-name' placed at any level (SETTINGS, globaly, Flow, Track, or Step) where SETTINGS is the default, and Step being the most specific will override the others.

Frames

Any number 'Frames' will permenantly be part of the outer HTML of any rendered web page. Each successive Frame is embedded wherever a single EpicMvc HTML element <e-page /> occurs, and can also include any number 'Parts' and any amount of HTML. Frames are designated in the Controller (app.coffee) using SETTINGS: frames:. The hash string is used to sort the order of the frames. For example the Dev packages uses frames: MMM_dev: 'bdevl'. This allows developers to place another frame either before or after, using e.g. AAA_something: 'frame-name' or ZZZ_something. The file would be found as Layout/*.html. Although the e-page element is used, it refers to either the next Frame, or lastly, the Layout.

Here is visually how EpicMvc will glue together all of the pieces:

Layout/update_your_app.html
Layout/main.html
Page/example.html
Part/header.html
Part/body.html
Part/footer.html

Example

The Frame HTML for Layout/update_your_app.html would be: 

	<div style="text-align: center; margin: 10px;">Time to Update your app</div>
	<e-page />

The Layout HTML for Layout/main.html would be: 

	<div class="container-fluid">
		<e-page />
	</div>

The Page HTML for Page/example.html would be: 

	<div class="row">
		<e-part part="header" />
	</div>
	<hr>
	<div class="row">
		<e-part part="body" />
	</div>
	<hr>
	<div class="row">
		<e-part part="footer" />
	</div>

The Part HTML for Part/header.html could be: 

	<p>Header Info</p>

The Part HTML for Part/body.html could be: 

	<p>Hello World!</p>

The Part HTML for Part/footer.html could be: 

	<p>Footer Info</p>

The following would render:

Time to Update your app

Header Info

Hello World!

Footer Info
