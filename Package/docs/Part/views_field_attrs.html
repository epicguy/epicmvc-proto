Field Attributes

Attribute Required Example Description
db_nm yes 'username' The database column name or REST endpoint hash name
type yes 'textarea' The type of the form field to display. See below for options.
label no 'Confirm Password' The label to place on a form field
req no true Field must not be blank if set to true
req_text no 'Password is required' The text to display if the field is required, but left empty
validate no 'regexp' The name of the validation filter to run. See below for options.
validate_expr no '[0-9]{3,4}' An expression passed to the validation filter
issue_text no 'Password must be at least 6 characters long' The text to display if there is an error validating the field
h2h no 'trimspaces:lower_case' Html-to-html filter specification. Applied to this field after the user has entered text to "clean" up the diplay. Can be array of filter names, or as a colon or comma separated list. See below for options.
d2h no 'phone' Data-to-Html translation: The name of the D2H filter to run. See below for options.
d2h_expr no 'international' An expression that gets passed to the D2H filter
h2d no 'date' Html-to-data translation: The name of the H2D filter to run. See below for options
h2d_expr no 'yyyy-mm-dd' An expression that gets passed to the H2D filter
cdata no [1,2,3,4,5,6,7] Control-data: Data that is control specific (ex: to populate multiple choice field values when type=pulldown:array)

Field Attribute Options

type text Single line text box
textarea Multi-line textbox
radio:array Radio buttons populated with the array from cdata attribute
radio:custom Radio buttons populated by the Model's fistGetChoices() method
yesno Checkbox - cdata contains checked and unchecked values
pulldown:array Pulldown populated by an array of data from the cdata attribute
pulldown:custom Pulldown populated by the Model's fistGetChoices() method
pulldown:wist Pulldown populated by the Wist Model; See cdata contains 'Wist_table_name:value_column:option_column'.
cdata [val1, val2,...] Used by pulldown:array and radio:array. List of values that will be used as the db value and display in the HTML. With this layout val1 will be both the first option displayed, and the value posted.
[[val, opt],[val2,opt2]...] Used by pulldown:array and radio:array. List of lists. Each sub-list will have a val that is used for the db, and an opt to display in the HTML
'Wist_table_name:value_column:option_column' Used with pulldown:wist type. Epic will call the 'Wist' Model with a table name of 'Wist_table_nm' and use the 'value_column' and 'option_column' of the Wist data to populate the options and values for a multiple choice input.
[ 'yes-value', 'no-value' ] Used with yesno type. EpicMvc will assign the 'yes_value' when the checkbox is checked. Defaults to [ '1', '0'].
validate test Validates any regular expression placed into validate_expr as String '^[0-9]{10}$' or RE /^[a-z_]+$/
email [Proto] Validates a valid email format
h2h pre Always run on each field, no-op in Base package; Proto provides: E.fistH2H$pre= (field,val) -> val.replace /[<>]/g, ''
trim [Proto] E.fistH2H$trim= (field,val) -> (String val).trim()
lower [Proto] E.fistH2H$lower= (field,val) -> (String val).toLowerCase()
upper [Proto] E.fistH2H$upper= (field,val) -> (String val).toUpperCase()
zero [Proto] E.fistH2H$zero= (field,val) -> val ? 0
null [Proto] E.fistH2H$null= (field,val) -> val ? null
empty [Proto] E.fistH2H$empty= (field,val) -> val ? ''
digits [Proto] E.fistH2H$digits=(field,val) -> val.replace /[^0-9]/g, ''
chars [Proto] E.fistH2H$chars=(field,val) -> val.replace /[^a-z]/ig, ''
d2h
sliceIt [Proto] E.fistD2H$sliceIt= (field, val) -> expr= field.d2h_expr; (String val).slice expr[ 0], expr[ 1]
h2d
zero [Proto] E.fistH2D$zero= (field,val) -> val ? 0
upper [Proto] E.fistH2D$upper= (field,val) -> (String val).toUpperCase()
