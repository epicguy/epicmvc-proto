Links and Navigation

This section explains how to make clickable links throughout your app. Links that are outside of forms, can be hyperlinks on text or on images. For these, we use the tag <epic:link_action>. The HTML must be aware of the values needed to process an action, and provide these on the tag along with an action label. If you must pass data in a form, you will instead need to use a form button within a form, so that the form data will be posted by the browser. This requires a different tag <epic:form_action> and this is documented in the next section called Forms, form fields, and buttons.

The <epic:link_action> tag does not require you to specify which page to display next. The only information needed is (a) a unique name for this action used later to connect to the business logic, and (b) any values to pass to that action. Suppose you wish to create a link in the current application, which will log the user out. This may be done like this: <epic:link_action action="logout" /> - the app.coffee file will contain a CLICK entry that maps your action called 'logout' to the proper JavaScript logic module used to effect a logout. Additionally, the app.coffee file will determine which page would naturally be displayed next, after this occurred.

Typically, a link appears inside an <epic:foreach> loop, such that each link is supposed to reference a different row of data. For example, suppose you have a list of people to send an email to, and the link is designed to send an email to just one of those addresses. A combination of the <epic:foreach> and an <epic:link_action> would be used, however this time a parameter must be included, which names the recipient. The HTML might look like this: 

<h3>Send email to one of the following:</h3>
<epic:foreach table="ListMaintainer/Recipients">
	<epic:link_action action="send_email" p:recipient="&Recipients/email;">&Recipients/email;</epic:link_action>
</epic:foreach>

Via data attributes

<h3>Send email to one of the following:</h3>
<epic:foreach table="ListMaintainer/Recipients">
	<a data-action="click:send_email" data-params="{'recipient':'&Recipients/email;'}">&Recipients/email;</a>
</epic:foreach>

Later in the app.coffee file, there would be instructions such as:

CLICKS:
	send_email:	call: 'ListMaintainer/send', use_fields: 'recipient', RESULTS: [
		{ r:{}, go: 'home/main/send_confirm' }
	]
