Forms, Form Fields, and Buttons

Forms are created using a standard <form> tag. The HTML for Form Fields and thier attributes are created using the tag: <epic:form_part form="FormName">. By default, EpicMvc will attempt to draw each field defined for the form using a generic HTML part that is included with EpicMvc.

Within a form, the clickable buttons are created by using the tag <epic:form_action action="my_action">, or via data attributes on a valid HTML element using the syntax: data-action="click:my_action:true".

Assuming you have created a form called 'Login' with any number of fields, and wish to create a button ("Login") the HTML might look like this:

login.page.html
<form>
	<div class="container">
		<epic:form_part form="Login"/>
	</div>
	<epic:form_action action="login" 	value="Login" class="btn btn-primary" /> <!-- via Tag -->
	<button data-action="click:login:true" value="Login" class="btn btn-primary" /> <!-- via data attributes -->
</form>

Notes: <form> method will default to POST.

To create a form and form fields or edit attributes of those fields, see the topic: Creating Forms and Editing FISTs.

Custom Form HTML

Custom HTML for a Form can be used by specifying the HTML View Part within the attribute: part="" on the <epic:form_part/> tag. When using a custom View Part to display a Form, EpicMvc will use the "Tag" Model to populate the Field information. The "Tag" Model exposes 'Control' and 'Field' tables that can be looped on to display every Control or Field that is defined for that Form.

Using a custom View Part to display the Form, the above example could be written as follows:

Example

login.page.html
<form>
	<epic:form_part form="Login" part="fist_login"/> <!-- uses the part called fist_login.part.html -->
	<epic:form_action action="login" value="Login" class="btn btn-primary" />
</form>
fist_login.part.html
<epic:foreach table="Tag/Field">
	<div>
		<epic:foreach table="Field/UserName">
			<input type="&UserName/type;"
				name="&UserName/name;"
				value="&UserName/value;"
				placeholder="&UserName/label;">
		</epic:foreach>
	</div>
	<div>
		<epic:foreach table="Field/Password">
			<input type="&Password/type;"
				name="&Password/name;"
				value="&Password/value;"
				placeholder="&Password/label;">
		</epic:foreach>
	</div>
</epic:foreach>
