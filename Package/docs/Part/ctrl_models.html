MODELS

MODELS is a top level key of the Controller structure and is used to define the Models (JavaScript Business Logic Classes) that are to be used in the application.

Example

E.app$myPkg=
	MODELS:
		League:		class: 'League',	inst: 'mpLg', fists: 'EditPlayer,AddPlayer'
		Wist:		class: 'Wist',		inst: 'mpW'
		Self:		class: 'Self',		inst: 'mpU'
		User:		class: 'Self',		inst: 'mpU' # Alias for Self Model

Define a Model

A Model is made available by defining it as a key under MODELS. This key is the name that will be used by the View and Controller when referencing the Model e.g League or User. The class: attribute is the name of the Javascript Class in code where the business logic resides, and the inst: attribute is the name of the instance created after calling new on the Javascript class.

Associate Forms (Fists) to a Model

Forms (actually a 'list of fields' called a 'fist') are associated with a Model by using the fists: attribute and declaring the names of the Fists in a comma separated string. When a form needs to be posted or populated, the controller will ask the Model associated with the form to do the work.

Creating a Model Alias

It is possible to create an Alias name for a Model. In the example above, User is an alias for Self, where both refer to the same instance of the 'Self' class.

Built-in EpicMvc Models

EpicMvc has a few built-in Models that are made available to you. They are defined in the Base Package app.coffee file and are merged in to the App Controller at run time. 

E.app$Base=
	MODELS:
		App:  class: "App$Base",  inst: "iBaseApp"		# Exposes controller values to the Views, and actions to control page-flow
		View: class: "View$Base", inst: "iBaseView"		# Implements the logic associated with all the special html tags and vars
		Fist: class: "Fist$Base", inst: "iBaseFist"		# 'Field list' business logic for e.g. 2-way binding and validation
		Tab:  class: "Tab$Base",  inst: "iBaseTab"		# View-state for Bootstrap type HTML things, such as Tabs, dropdowns, popups
		Wist: class: "Wist$Base", inst: "iBaseWist"		# 'Word list' model to read a static file of data and expose as view-tables
