MACROS

MACROS are simply shortcuts or aliases that are defined in your 'app' file to simplify programming. They have the exact same syntax and follow the same rules as 'ACTIONS'. Macros are especially useful if you have to perform the same logic in multiple places throughout the application. They are defined once at the top of the 'Controller' and can be referenced as a do: (wihtout a period) anywhere a Model.action can be called. Not using a period (dot) indicates this is a macro and not a Model.action call.

Example

E.app$docs=
	ACTION:
		...
	MODELS:
		...
	MACROS:
		start: do: 'User.check', next: [
			{ when:{ valid: 'no'},	go: 'logged_out//' }
			{ when:{ valid: 'yes'}, go: 'logged_in//' }
		]
		go_billing_select: do: 'Billing.select_plan_default', go: 'myprofile/bill/select'
		close_modal: do: 'Pageflow.restore_path'
	FLOWS:
		...
		TRACKS:
			...
			STEPS:
				...
				privacy: page: 'privacy_policy', ACTIONS:
					close_modal:	do: 'close_modal'
					cancel:			do: 'close_modal'
				terms: page: 'terms_of_use', ACTIONS:
					close_modal:	do: 'close_modal'
					cancel:			do: 'close_modal'
				...
