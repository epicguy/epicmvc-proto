Templates, Pages, and Parts

'Templates', 'Pages', and 'Parts' are HTML files located on disk, that contain valid HTML excluding the traditional <!DOCTYPE html>,<html>, <head>, <meta> and <body> tags. 'Templates' are saved with the extension .tmpl.html, 'Pages' are saved with the extension .page.html and 'Parts' are saved with the extension .part.html.

Parts

EpicMvc 'Parts' are the most basic components of the 'View'. A 'Part' could be as simple as a single <p> tag, or include most of the HTML for the application. However, the idea is to design 'Parts' as meaningful and modularized pieces of the HTML that can be included or looped on elsewhere in the 'View'.

To include the HTML defined in the 'Part' simply use the tag <epic:page_part part="part_file_name">. The 'part=' attribute is the name of .part.html file on disk. EpicMvc will then replace that tag with the actual HTML from the Part during render. The 'Part' tag can be included in any 'Page', 'Template' or even another 'Part'.

Pages

An EpicMvc 'Page' is the main component of the 'View'. Only one 'Page' is displayed at a time, and it is composed of HTML and any 'Parts' that are included using the page_part tag.

Templates

An EpicMvc 'Template' is HTML that includes a single EpicMvc 'Page' using the HTML tag <epic:page /> along with any other 'Parts'.

Here is visually how EpicMvc will glue together all of the pieces:

main.tmpl.html
example.page.html
header.part.html
body.part.html
footer.part.html

Example

The Template HTML for main.tmpl.html would be: 

	<div class="container">
		<epic:page />
	</div>

The Page HTML for example.page.html would be: 

	<div class="row">
		<epic:page_part part="header" />
	</div>
	<hr>
	<div class="row">
		<epic:page_part part="body" />
	</div>
	<hr>
	<div class="row">
		<epic:page_part part="footer" />
	</div>

The Part HTML for header.part.html could be: 

	<p>Header Info</p>

The Part HTML for body.part.html could be: 

	<p>Hello World!</p>

The Part HTML for footer.part.html could be: 

	<p>Footer Info</p>

The following would render:

Header Info

Hello World!

Footer Info
