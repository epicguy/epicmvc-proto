Tables: Accessing Dynamic Data in Tables Using the "foreach"

Most dynamic data is provided as multiple rows of values arranged in columns. These are accessed row-by-row using the <epic:foreach table="Class/Table">...</epic:foreach> tag.

Column values then are referenced in the content using &Table_name/column_name;

Many times a particular column value is really a nested table itself.

The nested table is accessed using: <epic:foreach table="Table_name/column_name_of_nested_table"> and value are accessed using &column_name_of_nested_table/column_name;. This can be done to any depth.

Here is an example of accessing the shipping rows of an order, and within each shipping row, the item rows, and within these items, any optional component information (such as doll attributes on a custom doll):

<epic:foreach table="OrderStatus/Shipment">
	<div class="shipments">Shipment &Shipment/counter; to:  &Shipment/first_name; &Shipment/last_name;</div> 
	<epic:foreach table="Shipment/Items">
		<div class="items">Item &Items/counter;: &Items/description;</div>
		<epic:foreach table="Items/Components" alias="C">  # Here is an example of the use of an "alias" to shorten names for reuse.
			<div class="components">Components &C/counter;: &C/description;</div>
		</epic:foreach> 
	</epic:foreach>
</epic:foreach>
