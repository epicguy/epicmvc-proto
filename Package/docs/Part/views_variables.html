Variables: Accessing Dynamic Data in Lists

Outputing or using variables on pages/parts or in code is really accessing dynamic data in what Epic calls "Lists".

When variables are output in the code, they are scalar string values. However when used in other epic functions, they can be type specific (number, string, bool, etc); subject to the rules of JavaScript.

To reference dynamic data in a 'List', you use three words separated by slashes, and preceded by an ampersand, and terminated with a semicolon. These rules are designed to allow you to insert a data value anywhere inside the content or markup in the HTML. The three parts of a list-data-value, are 'view-name', 'list-set-name' and 'list-item-name'. These are placed as follows: &view-name/list-set-name/list-item-name;

Example: Suppose you want to display today's date.

view-name
You first need to know what view-name to use. A veiw-name is linked with the JavaScript 'class' that makes dynamic data available to the HTML. We can use the sample 'Simple' application, which has the view-name of  'Simple'.
list-set-name
Next, you need the name of the 'set' of data values you are interested in. Typically, the JavaScript developer will create online documentation showing you which list-set-names are available for a give view-name. In our case, the list-set-name is 'General' to describe a set of general values made available by this JavaScript class.
list-item-name
Within each list-set, are a set of items by name. You choose an item by using its name as the third 'word' in a list-set reference. In our case, the developer has called "today's date" value: "date".

Putting it all together

Now, we can place this data list value, into our content; in our HTML:

Today is &Simple/General/date;

Advanced formatting

There are functions defined by the JavaScript programmers which can format a data value, for use in various places in the HTML. Suppose that a value is going to be used in a query, but might contain spaces, ampersand, equal sign, etc.:

<a href="/search.html?keywords=&Search/terms/last_keywords;">Link</a>

The value needs to be formatted for use as a value inside the URL. Assuming the programmer created the function needed, the HTML would specify instead:

<a href="/search.html?keywords=&Search/terms/last_keywords#url(query_value);">Link</a>
