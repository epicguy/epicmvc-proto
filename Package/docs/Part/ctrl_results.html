The next: when: flow logic

When a 'Model' is called with an action, the 'Controller' might be interested in the result and want to perform additional 'Model' actions or modify the pageflow based on the result. This is accomplished by using the next:[ {when: }... ] attribute when calling a Model. When a 'Model' has completed an 'action', EpicMvc will return a result object back to the 'Controller'. The 'Controller' can then look for specific values attached to this result object to decide what should happen next.

The next: attribute is an array of instructions, where each instruction contains the key when: which in turn has a set of key/value pairs defined. The 'Controller' will walk through the array attempting to find a matching instruction for the result object that came back from the 'Model' based on the key/value pairs defined in 'when:' for an instruction. Once an insruction is found to have a match, it will execute that line as if it were an 'ACTION'. If when: is missing or is defined as an empty object or the string 'default', the 'Controller' will execute that instruction no matter what the result of the Model is when it reaches this line. If it is set to a string value, it is compared to success: or ok: (whichever exists.) Otherwise each hash value must match the corresponding response hash value.

A next: line object can contain go: commands as well as do: fist: clear: set: etc. commands and can contain nested 'next:' arrays. Using this method, the 'Controller' can execute multiple 'ACTIONS' chained together on behalf of the User without the User having to do any work at all. When only one line exists, next: can be just that line object w/o the array, i.e. next: when: true, go: '//main'

Example

   ACTIONS:
      login: do: 'User.', fist: 'Login', next: [
         { when:{success:'SUCCESS'},                     go: 'logged_in//' }    # Or... when: 'SUCCESS'
         { when:{success:'FAIL', is_registered: 'NO'},   go: 'logged_out/sign_up/'}
         { when:{success:'FAIL', is_registered: 'YES'},  go: 'logged_out/forgot_password/'}
      ]
      logout: do: 'User.', next: [           # Could also be... logout: do: 'User.' next: go: 'logged_out//'
         { when:{}, go: 'logged_out//' }
      ]
