Dynamic Data: Accessing Model View Tables

Outputting dynamic, variable data within the 'View' is done by accessing 'Tables' made available through a 'Model'. The 'View' simply has to reference the information defined in a 'Table' and whenever the 'Model' changes the data, EpicMvc will re-draw the page, and the new data will be displayed.

A 'Model' will typically expose it's data as either a single row or a multiple row 'Table'. A 'Model' even has the ability to expose a 'Nested Table' within any row's value, to any depth. This limited structure makes it easy for View designers to reason about, yet is as flexible as a Relational DB schema. There are countless possibilities of how Model Tables can be designed, so review the documentation for a specific Model to see what data is available to the view.


Single Row Table

A single row 'Table' is typically designed so that the View can quickly and easily access it's data. Because there is only one row in the table, the 'View' can simply specify 3 things to access the data: The name of the 'Model', the name of the 'Table', and the name of the 'Column' of data interested in. The syntax used by the 'View' is these three names separated by slashes, preceded by an ampersand, and terminated with a semicolon as follows: &ModelName/TableName/colName;


Example
<p>
	Team Name: &League/Options/selected_team_name; <br>
	Team Owner: &League/Options/selected_team_owner; <br>
	Place: &League/Options/current_place; <br>
</p>

Team Name: Kershawshank Redemption
Team Owner: Joe Smith
Place: 8th


Multiple Row Table

When data is exposed using a multiple row Table, EpicMvc provides the <e-foreach table="ModelName/TableName"> tag to draw the HTML content defined inside the tag for each row of the Model Table in the table= attribute. This is a convienient way to define the HTML once for a single row, but render it multiple times with dynamic data for each row.

Once inside the <e-foreach> tag, row data can be accessed by using the syntax: &TableName/colName;

Example
<table class="table table-condensed table-striped">
	<tr><th>Team</th><th>Points</th></tr>
	<e-foreach table="League/Standing">
		<tr><td>&Standing/team_name;</td><td>&Standing/points;</td></tr>
	</e-foreach>
</table>

Nested Sub-Table

Tables can be nested inside a row. To access the Nested Table, simply use another <e-foreach> tag inside the outer 'e-foreach' tag and set the table attribute using the syntax: table="OuterTable/InnerTable".

Example
<table class="table table-condensed table-striped">
	<tbody>
		<tr><th>Team</th><th>Player</th></tr>
		<e-foreach table="League/Roster">
			<tr>
				<td rowspan="3" class="col-md-3">&Roster/team_name;</td>
				<td> </td>
			</tr>
			<e-foreach table="Roster/Player">
				<tr>
					<td>&Player/first_name; &Player/last_name;</td>
				</tr>
			</e-foreach>
		</e-foreach>
	</tbody>
</table>

Using a Table Alias Name

An alias can be used within the <e-foreach> tag by using the attribute: alias="TableAlias". This is useful if the table name is long, or you would like to use a single page_part for different tables.

Example
<table class="table table-condensed table-striped">
	<tr><th>Team</th><th>Points</th></tr>
	<e-foreach table="League/Standing" alias="S">
		<tr><td>&S/team_name;</td><td>&S/points;</td></tr>
	</e-foreach>
</table>
